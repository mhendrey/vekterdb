VekterDB turns any SQLAlchemy compliant database into a vector database using the FAISS library to index the vectors.

class vekterdb.vekterdb.VekterDB(table_name: str, idx_name: str = 'idx', vector_name: str = 'vector', columns_dict: Dict[str, Dict] = {}, url: str = 'sqlite:///:memory:', connect_args: Dict = {}, faiss_index: str | None = None)[source]ÔÉÅ

Bases: object

VekterDB uses a minimum of two columns in the database table: idx_name (BigInteger, default ‚Äòidx‚Äô) and vector_name (LargeBinary, default ‚Äòvector‚Äô). Vectors are numpy arrays of np.float32, serialized to bytes using tobytes(), to comply with FAISS requirements.

Additional database table columns can be specified with columns_dict using SQLAlchemy‚Äôs Column arguments. For example, include a unique, indexed id field (str) and a non-unique product_category field (str) with:

my_db = VekterDB(
    "my_table",
    columns_dict={
        "id": {"type": Text, "unique": True, "nullable": False, "index": True},
        "product_category": {"type": Text},
    }
)
Parameters:

  • table_name (str) ‚Äì Database table name, either existing or new.

  • idx_name (str, optional) ‚Äì Column name that stores the FAISS index integer ID and is the primary key for the database table. It must be unique and consecutive from [0, n_records). The default name is ‚Äúidx‚Äù.

  • vector_name (str, optional) ‚Äì Column name that stores the vector information. Default is ‚Äúvector‚Äù.

  • columns_dict (Dict[str, Dict], optional) ‚Äì

    Names (key) of additional columns to include in the table. The values are arguments that will be passed to SQLAlchemy‚Äôs Column. Default is {}.

    When connecting to an existing database table, this argument is not necessary.

  • url (str, optional) ‚Äì URL string to connect to the database. Passed to SQLAlchemy‚Äôs create_engine. Default is ‚Äúsqlite:///:memory‚Äù; an in-memory database.

  • connect_args (Dict, optional) ‚Äì Any connection arguments to pass to SQLAlchemy‚Äôs create_engine. Default is {}.

  • faiss_index (str, optional) ‚Äì If given, then load an existing FAISS index saved by that name. Default is None.

create_index(faiss_index: str, faiss_factory_str: str, metric: str = 'inner_product', sample_size: int = 0, batch_size: int = 10000, use_gpu: bool = False, faiss_runtime_params: str | None = None)[source]ÔÉÅ

Create a FAISS index and save to disk when completed.

Parameters:

  • faiss_index (str) ‚Äì Name of the file to save the resulting FAISS index to.

  • faiss_factory_str (str) ‚Äì FAISS index factory string, passed to faiss.index_factory()

  • metric (str, optional) ‚Äì Metric used by FAISS to determine similarity. Valid values are either inner_product or L2. Default is inner_product

  • sample_size (int, optional) ‚Äì Number of vectors to sample from the database table to train the FAISS index. If 0, then all vectors are used. Default is 0.

  • batch_size (int, optional) ‚Äì Number of vectors to add into the index at one time. Also passed to sample_vectors() to specify number of vectors to pull back from the database table at a time. Default is 10,000.

  • use_gpu (bool, optional) ‚Äì Whether to use GPU(s). Default is False. NOTE: Not implemented yet

  • faiss_runtime_params (str, optional) ‚Äì Set FAISS index runtime parameters before adding vectors. Likely only useful if using a quantizer index (e.g., IVF12345_HNSW32). The quantizer index (HNSW32) will be used during the index.add() to determine which partition to add the vector to. You may want to change from the default, whether that is the FAISS default (efSearch=16) or the value saved in self.faiss_runtime_parameters. ‚Äúquantizer_efSearch=40‚Äù would be an example value for the example index given. If used, self.faiss_runtime_parameters is set back to its value before function invocation. Default is None

Raises:

  • FileExistsError ‚Äì If a FAISS index is already assigned to this table.

  • FileExistsError ‚Äì If the file faiss_index already exists on disk.

  • TypeError ‚Äì If the metric is not either ‚Äúinner_product‚Äù | ‚ÄúL2‚Äù

nearest_neighbors(fetch_column: str, fetch_values: List, k_nearest_neighbors: int, *col_names: str, k_extra_neighbors: int = 0, rerank: bool = True, threshold: float | None = None, search_parameters=None, batch_size: int = 10000) → List[Dict][source]ÔÉÅ

Find the nearest neighbors of the query records in the table based on vector similarity. Optionally keep only neighbors whose similarity exceeds the threshold.

Parameters:

  • fetch_column (str) ‚Äì Column in the database table to use for query record retrieval.

  • fetch_values (List) ‚Äì Values to match in the fetch_column.

  • k_nearest_neighbors (int) ‚Äì Number of nearest neighbors to return.

  • *col_names (str) ‚Äì List of columns to use in the query and neighbor records. Default of None uses all columns.

  • k_extra_neighbors (int, optional) ‚Äì Extra neighbors to return from FAISS index before reranking. If using a vector quantizer (e.g., PQ), FAISS orders results based upon the estimated similarities which likely differs from the true similarities calculated here. Default is 0.

  • rerank (bool, optional) ‚Äì If True, rerank neighbors according to their true similarities. Otherwise the order is determined by the FAISS index‚Äôs index.search(). Default is True.

  • threshold (float, optional) ‚Äì Only keep neighbors whose similarities exceed the threshold. Default is None which keeps all neighbors returned.

  • search_parameters (faiss.SearchParameters, optional) ‚Äì Use these search parameters instead of the current runtime FAISS parameters. Passed to FAISS‚Äôs index.search(). See [FAISS documentation](https://github.com/facebookresearch/faiss/wiki/Setting-search-parameters-for-one-query)

  • batch_size (int, optional) ‚Äì Number of query records to retrieve from the database table at one time. Passed to fetch_records(). Default is 10,000.

Returns:

For each query, a dictionary containing the query‚Äôs record and a list of the its neighbors‚Äô records. A neighbor record includes the ‚Äúmetric‚Äù similarity.

Return type:

List[Dict]

Change FAISS runtime parameters with a human-readable string. Parameters are separated by commas. For example, with the index ‚ÄòOPQ64,IVF50000_HNSW32,PQ64‚Äô, you can use ‚Äúnprobe=50,quantizer_efSearch=100‚Äù to set both the nprobe in the IVF index and the efSearch in the HNSW quantizer index.
